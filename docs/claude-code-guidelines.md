# Claude Code 補助ガイドライン

**最終更新**: 2026-03-14

CLAUDE.md から分離した補助的なガイドライン。必要な時に参照する。

---

## 開発フロー（全体の流れ）

```
1. CLAUDE.mdの手順に従ってプロジェクト理解を深める
     ↓
2. issues/backlog.md から作業を選択
     ↓
3. draft-xxx.md に切り出し
     ↓
4. ユーザーの指示を受けて調査、ドラフトに反映
     ↓
5. 作業の許可を得る
     ↓
6. GitHub Issue 作成 → ローカルファイルを番号付きにリネーム
     ↓
7. masterで直接作業・実装
     ↓
8. 作業完了 → 作業ログ作成（dev-logs/）
     ↓
9. Claude Codeにレビュー依頼 → レビュー記録作成（reviews/）
     ↓
10. レビュー指摘対応（全指摘を以下のいずれかで解決）
    - 致命的/重要: その場で修正 or backlog追記
    - 軽微: backlog追記（対応任意）
    - 棄却: 人間が判断し、レビュー記録に理由を記載
     ↓
11. コミット（Closes #番号 で自動クローズ）→ push
```

---

## 開発ログガイドライン

開発タスク完了時：
1. `/dev-logs/` に `YYYY-MM-DD_task-description.md` 形式で作成
2. **重要**: 日付は必ず `date +%Y-%m-%d` コマンドで取得（手動入力禁止）
3. `DEVELOPMENT_LOG.md` インデックスを更新（新しいログを一番上に追加）
4. **機密情報は絶対に含めない**

**ログファイル構造**:
```markdown
# タスク名

**作業日**: YYYY-MM-DD
**GitHub Issue**: #番号（あれば）

---

## 概要
（何をしたかの要約）

## 実施内容
（詳細）

## 変更ファイル一覧
| ファイル | 変更内容 |
|----------|----------|
| xxx | 新規作成 |

## 次のアクション
（あれば）
```

**基本ルール**:
- 1コミット1作業ログが前提
- 変更ファイル一覧は必須

## よく使う指示への対応

| ユーザーの指示 | やること |
|---------------|----------|
| 「プロジェクトについて理解して」 | CLAUDE.mdの「開発準備手順」を実行（`/prep`） |
| 「バックログを整理して」 | `issues/backlog.md` の一時メモ欄を適切なカテゴリに振り分け |
| 「○○をイシュー下書きにしよう」 | `issues/draft-xxx.md` を作成、backlogから削除 |
| 「作業ログ作成して」 | `/create-devlog` スキルを実行 |
| 「レビューして」 | `/review` スキルを実行 |
| 「作業完了処理して」 | `/wrap-up` スキルを実行 |
| 「gitコミットに進みましょう」 | 作業ログ修正（必要なら）→ 関連ファイルをadd → コミット |
