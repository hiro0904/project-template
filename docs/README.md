# ドキュメント管理

**最終更新**: 2026-03-14

プロジェクトの設計書・仕様書・ガイドラインを管理するディレクトリ。

## ディレクトリ構成

```
docs/
├── README.md                    # 本ファイル（運用ルール）
├── claude-code-guidelines.md    # Claude Code 補助ガイドライン
├── archive/                     # 完了・廃止されたドキュメント
└── （プロジェクト固有の仕様書を追加していく）
```

## ドキュメント運用ルール

### 新規ドキュメント作成時
- 冒頭に `**最終更新**: YYYY-MM-DD` を記載
- 日付は必ず `date +%Y-%m-%d` コマンドで取得

### 更新タイミング
- 実装が完了した時（実装後にドキュメント更新）
- 仕様変更があった時
- バグ修正で動作が変わった時

**原則**: 実装 → 検証 → ドキュメント更新の順番。想像で書かない。

### 仕様書更新フロー
- API/機能追加・変更時 → 関連する仕様書を更新
- **作業ログ作成前に仕様書更新を完了させる**（フロー: 実装 → 仕様書更新 → 作業ログ）

### アーカイブ運用
- 完了した機能の旧仕様、廃止された設計書 → `docs/archive/` に移動
- アーカイブ移動時、元のファイル名に `_archived_YYYY-MM-DD` サフィックスを付ける
- アーカイブ理由を冒頭に追記する

```markdown
<!-- 例 -->
> **アーカイブ**: 2026-03-14 — v2設計に移行したため廃止

（元の内容）
```

### 鮮度チェック
- `/update-docs` スキルで定期的にドキュメントの鮮度をチェックできる
- 最終更新日とコードの最終コミット日を突き合わせて、更新漏れを検出する
