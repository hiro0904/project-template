# プレゼン資料自動生成システム

**最終更新**: 2026-04-23

## 概要

content プロジェクト（YouTube動画制作）で構築・運用していたスライド自動生成の仕組みを、汎用的に使えるようプロジェクトテンプレートに移行したもの。

ミーティングや発表の資料作成を、以下のワークフローで効率化する：

```
ローデータ収集 → 骨格作成 → 原稿作成 → スライド構成設計 → PPTX自動生成
```

## 何ができるか

- **ローデータ管理**: 音声入力メモ・リサーチ結果・アイデアを `work/raw/` に蓄積
- **原稿作成支援**: Claude Code との対話で骨格→本文を段階的に構築
- **スライド自動生成**: 原稿から pptxgenjs で PowerPoint ファイルを自動生成
- **18種類のテンプレート**: タイトル、目次、コンテンツ、カード、表、画像配置など

## 構成ファイル

```
project/
├── presentations/                  # 発表資料の管理
│   ├── README.md                   # 制作フローの説明
│   └── mtg-XXXX-テーマ名/          # 発表単位
│       ├── draft.md                # 骨格 + 原稿
│       ├── slide.md                # スライド構成書
│       └── work/raw/              # ローデータ
├── tools/pptx/                     # スライド生成ツール
│   ├── package.json                # pptxgenjs 依存
│   ├── templates/general.js        # スライドテンプレート（18種）
│   ├── scripts/                    # 発表ごとの生成スクリプト
│   └── output/                     # 生成先
├── .claude/commands/gen-slide.md   # /gen-slide スキル
└── docs/presentation-system.md     # 本ファイル
```

## セットアップ

```bash
cd tools/pptx
npm install
```

## 使い方

### 1. フォルダ作成

```bash
mkdir -p presentations/mtg-0001-週次報告/work/raw
```

### 2. ローデータを溜める

`work/raw/` に .md ファイルでメモ・リサーチ結果を保存。

### 3. 原稿作成

Claude Code に「原稿を作って」と依頼。`draft.md` に骨格→本文の順で作成される。

### 4. スライド生成

```
/gen-slide presentations/mtg-0001-週次報告/
```

Claude Code が以下を実行：
1. `slide.md`（スライド構成書）を作成 → ユーザー確認
2. `tools/pptx/scripts/mtg-0001.js`（生成スクリプト）を作成
3. スクリプトを実行して PPTX を出力

### 5. 確認・発表

PowerPoint で開いて確認。必要に応じて微調整して発表。
