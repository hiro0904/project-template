## 開発作業開始前の重要な方針

### ユーザー確認必須
具体的な開発・実装作業を開始する前に：
1. **必ずユーザーと方向性を確認する**
2. **実装仕様や設計についてユーザーの合意を得る**
3. **ユーザーが明示的に許可した場合のみ実装を開始する**

### 確認フロー
```
1. 作業内容の提案・説明
2. 実装方針・仕様の詳細説明
3. ユーザーからの確認・許可待ち
4. 許可後に実装開始
```

### 対話スタイル
- **AskUserQuestionツールは使用しない**
- **選択肢形式ではなく、議論ベースで進める**
- 仕様や方針を決める際は、調査結果と選択肢を**テキストで説明**し、ユーザーと対話しながら決定する

### 音声入力への対応
- ユーザーは音声入力（Aqua Voice）を使用している
- プロンプトに誤字・誤変換がある場合は、音声認識の誤検知として推測・解釈する
- 例：「Claude Code」→「クラウドコード」、技術用語の誤変換など

### 新規プログラム・仕様変更時の必須提案フロー
新しい処理を作成する時や既存仕様を変更する時は：
1. **必ず仕様を詳細に説明する**
2. **実装提案を明確に行う**
3. **ユーザーから明示的な許可を得る**
4. **許可後に修正・実装を開始する**

## 開発準備手順

開発タスクを開始する前に、`/understand` コマンドを実行してプロジェクトの理解を深めてください。
**auto-compact後は必ず実行すること。**

コマンド定義: `.claude/commands/understand.md`

### Bashコマンド実行ルール

- `git status`, `git log --oneline -5`, `git diff --name-only` は**個別のBash呼び出し**で実行する
- `&&` でチェーンしない（許可済みコマンドとして自動実行されるため）

### 作業対象の詳細理解（参照テーブル）

作業対象に応じて、必要なファイルだけ読む:

## ドキュメント作成ルール

### mdファイルの構成
- **冒頭に「最終更新: YYYY-MM-DD」を記載**
- 更新時は日付も更新する
- **日付は必ず `date +%Y-%m-%d` コマンドで取得**（手動入力禁止、ミス防止のため）

### README更新タイミング
- フォルダ構成が変わった時
- 新しいファイルを追加した時
- 運用ルールが変わった時

### 資料更新タイミング
- 実装が完了した時（実装後にドキュメント更新）
- 仕様変更があった時
- バグ修正で動作が変わった時

### 仕様書更新タイミング
- API/機能追加・変更時 → 関連する仕様書を更新
- **作業ログ作成前に仕様書更新を完了させる**（フロー: 実装 → 仕様書更新 → 作業ログ）

**原則**: 実装 → 検証 → ドキュメント更新の順番。想像で書かない。

## 開発ガイドライン

### コード品質

コードを含む作業の場合、作業終了前に適切な検証を実行する。

### Gitステージング検証必須

gitコミット前に：
1. `git add` 後に `git status` を実行
2. 意図したすべてのファイルが "Changes to be committed" にあることを確認
3. 関連ファイルが "Changes not staged for commit" に残っていないことを確認
4. 検証成功後のみコミットを実行

### コミットメッセージ
- `Co-Authored-By` は付けない

### git push 禁止
- **Claude Codeは `git push` を実行しない**（pushは人間が行う）
- pushするか聞く必要もない

## よく使う指示への対応

詳細は `docs/claude-code-guidelines.md` を参照。

### プレゼン資料関連

詳細なワークフローは `presentations/README.md` を参照。

| ユーザーの指示 | やること |
|---------------|----------|
| 「ヒアリングして」 | ユーザーの音声入力をそのまま `work/raw/hearing-XX.md` に記録。整形しない |
| 「リサーチして」「調べて」 | 調査してraw/にresearch-XX.mdとして保存。またはプロンプト集を作成 |
| 「骨格作って」 | raw/の内容をもとにdraft.mdに「伝えたいこと」+セクション構成を作成 |
| 「原稿作って」 | 骨格に沿って本文を執筆。draft.mdの後半に追記 |
| 「スライド作って」 | `/gen-slide` を実行。slide.md作成→確認→PPTX生成 |
| 「発表資料を準備して」 | presentations/README.mdのフローに沿ってPhase 1から開始 |
