## Claude Codeへの重要な方針

### DB直接アクセス禁止
- **Claude CodeはMySQLデータベースに直接アクセスしてはならない**
- **データベース調査が必要な場合は、SQL文をユーザーに提示してユーザーに実行を依頼する**

### DB調査の正しい手順
```
1. 必要な調査内容を説明
2. 実行してほしいSQL文を提示
3. ユーザーによる実行と結果共有を待つ
4. 結果に基づいて分析・提案を行う
```

### DBスキーマ変更時
- スキーマ変更コマンド（`prisma db pull` 等）実行前に必ずユーザー確認を求める

## ドキュメント構成の考え方

### 基本方針: 関連テーブルをまとめる

テーブル仕様書は **1テーブル1ファイルではなく、関連するテーブルをまとめて1ファイル** にする。

**理由**:
- 関連テーブルは一緒に見ることが多い
- リレーションが分かりやすい
- ファイル数が爆発しない

### 例: ユーザー管理ドメイン

```markdown
# ユーザー管理テーブル

## 概要
ユーザー認証・権限管理に関するテーブル群

## テーブル一覧
- users - ユーザー基本情報
- roles - 権限ロール
- user_roles - ユーザーとロールの紐付け

## 各テーブル詳細
（table-spec-template.md の形式で記載）

## リレーション図
（必要に応じて）
```

### ドメインの分け方

プロジェクトの業務領域に合わせて分ける。例：
- `user-management.md` - ユーザー・認証・権限
- `reservation.md` - 予約関連
- `inventory.md` - 在庫管理
- `master-data.md` - マスターテーブル群
