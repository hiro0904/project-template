# 大規模タスク管理

**最終更新**: YYYY-MM-DD

このフォルダは、1つのIssueファイルで管理できない大きなタスク・プロジェクトを管理するためのディレクトリです。

## いつ使うか

- 複数のIssueにまたがる大規模な移行作業
- 長期間にわたる設計検討
- 複雑な仕様検討が必要な機能開発

## 使い方

```
project/
├── README.md                 # 本ファイル
└── {project-name}/           # プロジェクトごとにフォルダを作成
    ├── README.md             # プロジェクト概要・ステータス
    ├── discussion.md         # 議論・検討の記録
    └── ...                   # その他必要なドキュメント
```

## 注意

- 小さなタスクは `issues/backlog.md` → GitHub Issue で管理
- このフォルダは必要に応じて使用（必須ではない）
