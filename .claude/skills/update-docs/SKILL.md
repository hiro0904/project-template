# ドキュメント鮮度チェック & 更新

## 手順

1. **対象ファイル検出**: `最終更新` フィールドを持つmdファイルを一覧化（issues/, dev-logs/, reviews/ は対象外）
2. **鮮度チェック**: 各ファイルの最終更新日と、対応ドメインの最終コミット日を突き合わせ
3. **要更新一覧を提示**: テーブル形式でユーザーに報告
4. **ユーザー許可後に更新**: 実態（コード・git履歴）に基づいて修正

## ドメイン対応関係

<!-- プロジェクトに合わせてカスタマイズする -->
| ドキュメントのパス | git logで確認するパス |
|---|---|
| `docs/*.md` | プロジェクトルート |
| `README.md` | プロジェクトルート |

## 注意

- 日付は `date +%Y-%m-%d` で取得
- 想像で書かない（実態に基づく）
- 大きな仕様変更が必要な場合はユーザーに相談
