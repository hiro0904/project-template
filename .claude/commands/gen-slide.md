原稿からスライド（PowerPoint）を生成してください。

## 入力

ユーザーから以下のいずれかが提供されます：
- プレゼンのパス（例: `presentations/mtg-0001-週次報告/`）
- 原稿ファイルのパス（例: `presentations/mtg-0001-週次報告/draft.md`）
- 引数なしの場合、どのプレゼンか確認する

## 事前準備（必須）

以下のファイルを読み込んでから生成に入ること：
1. 対象プレゼンの `draft.md` — 原稿の内容を把握
2. `tools/pptx/templates/general.js` — 利用可能なテンプレートパターンを確認
3. 過去の生成スクリプト（`tools/pptx/scripts/` 内）があれば1つ参考に読む

## 生成フロー

### Step 1: slide.md 作成（構成設計）

原稿を読み、以下の構成で `presentations/mtg-XXXX-テーマ名/slide.md` を作成する。

```markdown
# mtg-XXXX スライド構成

**最終更新**: YYYY-MM-DD

## メタ情報

- **原稿**: [draft.md](./draft.md)
- **生成スクリプト**: `tools/pptx/scripts/mtg-XXXX.js`
- **出力先**: `tools/pptx/output/`

---

## スライド一覧

| # | セクション | テンプレート | タイトル |
|---|---------|----------|--------|
| 1 | - | タイトル | ... |
| ... | ... | ... | ... |

---

## スライド詳細

### スライド 1 — タイトル
- テンプレート: タイトル
- メインタイトル: ...
- サブタイトル: ...
（各スライドの詳細を記述）
```

#### テンプレート選択の判断基準

| 内容 | テンプレート |
|------|----------|
| 発表の表紙 | タイトル |
| 全体の流れを示す | 目次 |
| 人物紹介 | 自己紹介 |
| 大セクションの開始 | セクション区切り |
| 端的なメッセージを強調 | メッセージ強調 |
| 2つのポイントを並べる | 2ポイントカード |
| 3つのポイントを並べる | 3ポイントカード |
| 対比・比較 | 2カラム |
| 見出し+説明の繰り返し | コンテンツ（箇条書き） |
| データ一覧 | 表 |
| 写真+説明 | 画像+テキスト |
| 写真メイン | 全面画像+オーバーレイ |
| 締め | エンディング |

テンプレートに当てはまらない場合は、ユーザーに新しいテンプレートの作成を提案する。

### Step 2: ユーザー確認

slide.md の内容をユーザーに提示し、確認を得る。
- スライド構成（枚数・順序）
- 各スライドのテンプレート選択
- 内容の過不足

**ユーザーの許可を得てから Step 3 に進む。**

### Step 3: 生成スクリプト作成

`tools/pptx/scripts/mtg-XXXX.js` を作成する。

#### スクリプトの構成ルール
- `pptxgenjs` を使用（`tools/pptx/node_modules` を参照）
- カラーパレットは `tools/pptx/templates/general.js` の定義に準拠
- フォントは `游ゴシック`
- ヘルパー関数（`addContentHeader`, `addBulletContent`, `addCardSlide`, `addSectionSlide`）を定義して再利用
- 出力先: `tools/pptx/output/` に出力

#### コンテンツスライドの統一ルール
- 見出し: 14pt 太字 deepSpace
- 本文: 12pt yaleBlue
- 見出し間の空行: fontSize 12
- キーメッセージ: 13pt 太字 deepSpace（各スライドで一番伝えたいことを1行）

### Step 4: 実行・出力

```bash
cd tools/pptx && node scripts/mtg-XXXX.js
```

出力先ディレクトリが存在しない場合は事前に作成する。

## 出力

生成後、以下を報告：
- 出力ファイルのパス
- スライド枚数
- 「PowerPointで開いて確認してください」

修正が必要な場合は、slide.md とスクリプトを更新して再生成する。
